# 让 Agent 会"说话"：消息发送

## 📌 什么是消息发送？

消息发送是 Agent 与用户交互的核心能力。OpenClaw 支持多种消息渠道，让 Agent 随时随地"说话"。

---

## 🎯 支持的消息渠道

### 内置渠道
- **WhatsApp** - 全球最常用
- **Telegram** - 开发者最爱
- **Discord** - 社区/游戏
- **iMessage** - Apple 生态

### 插件渠道
- **Mattermost** - 企业协作
- **Feishu/钉钉** - 中国企业
- **更多...** - 社区开发中

---

## 🔧 消息发送的三种方式

### 1️⃣ 直接回复

**场景：** 用户提问 → Agent 回答

**实现：**
```javascript
// OpenClaw 自动处理
用户："今天天气如何？"
Agent："今天晴天，气温 25°C"
```

**特点：**
- 最简单的交互模式
- OpenClaw 自动路由
- 无需额外配置

---

### 2️⃣ 主动发送

**场景：** 定时提醒、告警通知

**实现：**
```bash
# 使用 message 工具
openclaw message send \
  --target telegram \
  --message "提醒：30 分钟后有会议"
```

**特点：**
- Agent 主动发起
- 需要目标渠道配置
- 适合定时任务

---

### 3️⃣ 富媒体消息

**场景：** 发送图片、文件、语音

**实现：**
```bash
# 发送图片
openclaw message send \
  --target telegram \
  --media /path/to/image.jpg \
  --caption "这是今天的截图"

# 发送文件
openclaw message send \
  --target telegram \
  --file /path/to/report.pdf
```

**特点：**
- 支持多种媒体类型
- 不同渠道有不同限制
- 注意文件大小限制

---

## 💡 最佳实践

### 1. 渠道选择

| 场景 | 推荐渠道 | 理由 |
|------|----------|------|
| 个人通知 | Telegram | 免费、API 友好 |
| 企业协作 | Feishu/钉钉 | 集成办公系统 |
| 社区运营 | Discord | 频道管理方便 |
| 大众用户 | WhatsApp | 用户基数大 |

### 2. 消息格式

**好的消息：**
```
【重要】服务器告警
时间：2026-03-12 08:00
问题：CPU 使用率 > 90%
建议：检查进程，考虑扩容
```

**不好的消息：**
```
服务器有问题，CPU 很高，快看看
```

### 3. 频率控制

- 避免消息轰炸
- 重要消息才主动发送
- 提供"免打扰"选项

## 🔧 实战示例

### 示例 1：定时提醒

```javascript
// 技能脚本
#!/bin/bash
MESSAGE="提醒：每天 20:00 发布文章"
openclaw message send \
  --target telegram \
  --message "$MESSAGE"
```

### 示例 2：告警通知

```javascript
// 监控脚本
if [ $CPU_USAGE -gt 90 ]; then
  openclaw message send \
    --target telegram \
    --message "【告警】CPU 使用率：${CPU_USAGE}%"
fi
```

### 示例 3：日报发送

```javascript
// 日报脚本
DAILY_REPORT=$(generate_report)
openclaw message send \
  --target telegram \
  --message "$DAILY_REPORT"
```
