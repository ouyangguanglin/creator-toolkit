## 什么是技能？

简单说，技能就是教 AI 怎么用工具的说明书。

有了技能，你的 Agent 可以：
- 用 GitHub CLI 管项目
- 查天气 API 告诉你带不带伞
- 自动发消息到 Telegram、Discord、飞书
- 控制浏览器干活

这些能力不是自带的，是靠技能系统加进去的。

---

## 技能加载的三层优先级

OpenClaw 从三个地方加载技能，优先级从高到低：

```
1. 工作区技能 (~/.openclaw/workspace/skills)     ← 你自定义的，优先级最高
2. 本地管理技能 (~/.openclaw/skills)             ← 所有 Agent 共享
3. 内置技能 (OpenClaw 安装包自带)                 ← 优先级最低
```

**什么意思？** 如果你在 workspce 里放一个同名的技能，会覆盖 OpenClaw 自带的版本。这个设计让你能定制任何行为。

---

## 创建第一个技能

### 1. 建目录

```bash
mkdir -p ~/.openclaw/workspace/skills/hello-world
```

### 2. 写 SKILL.md

每个技能的核心是一个 `SKILL.md` 文件：

```markdown
---
name: hello_world
description: 一个简单的问候技能
---

# Hello World 技能

当用户要求打招呼时说"你好"。

## 回复示例

- 标准："你好！有什么可以帮你的？"
- 早晨："早上好！"
- 晚上："晚上好！"
```

### 3. 测试

```bash
# 重启 Gateway
openclaw gateway restart

# 或者直接让 Agent 刷新
"刷新一下技能列表"
```

## 实战：GitHub 技能是怎么写的

这是工作区里真实存在的技能，我拆解给你看：

```markdown
---
name: github
description: 使用 gh CLI 与 GitHub 交互
---

# GitHub 技能

## 检查 PR 的 CI 状态

```bash
gh pr checks 55 --repo owner/repo
```

## 列出最近的 workflow 运行

```bash
gh run list --repo owner/repo --limit 10
```

## 查看失败日志

```bash
gh run view <run-id> --repo owner/repo --log-failed
```

## 高级查询用 API

```bash
gh api repos/owner/repo/pulls/55 --jq '.title, .state, .user.login'
```
```

**关键点：**
- 直接给命令，别说废话
- 用真实参数，别用 `<placeholder>`
- 展示常见场景

## 技能门控：条件加载

技能可以设置加载条件，不满足就不激活：

```markdown
---
name: weather-pro
description: 查天气
metadata:
  {
    "openclaw": {
      "requires": {
        "bins": ["curl"],           # 系统必须有 curl 命令
        "env": ["WEATHER_API_KEY"], # 必须有这个环境变量
        "config": ["weather.enabled"] # openclaw.json 里必须开启
      }
    }
  }
---
```

**常见门控条件：**

| 条件 | 说明 | 例子 |
|------|------|------|
| bins | 必须有这些命令 | `["ffmpeg", "yt-dlp"]` |
| anyBins | 至少有一个就行 | `["chrome", "brave"]` |
| env | 环境变量 | `["API_KEY"]` |
| config | 配置项 | `["browser.enabled"]` |
| os | 操作系统 | `["darwin", "linux"]` |

## 配置技能：openclaw.json

在 `~/.openclaw/openclaw.json` 里控制技能：

```json5
{
  skills: {
    entries: {
      "weather-pro": {
        enabled: true,  // 禁用就改成 false
        env: {
          WEATHER_API_KEY: "你的 key"
        }
      }
    }
  }
}
```

## ClawHub：技能市场

ClawHub (https://clawhub.com) 是 OpenClaw 的技能商店。

```bash
# 搜索
clawhub search weather

# 安装
clawhub install weather

# 更新全部
clawhub update --all
```

**查找顺序：** skillhub (CN 优化) → clawhub (公共)

## Token 消耗

技能列表会进系统提示，消耗 Token：

- 基础开销：195 字符
- 每个技能：约 97 字符 + 名称描述
- 估算：每个技能约 24 Token

10 个技能大概 240 Token，不算多，但也没必要装一堆不用的。

---

## 安全问题

第三方技能是别人写的代码，要注意：

1. **先看再装** — 读 SKILL.md，看有没有可疑命令
2. **API Key 别硬编码** — 用 openclaw.json 的 env 注入
3. **沙箱不是万能的** — 容器里也得装对应命令

## 实战项目：写个天气技能

### 1. 建目录

```bash
mkdir -p ~/.openclaw/workspace/skills/weather-pro
```

### 2. 写 SKILL.md

```markdown
---
name: weather-pro
description: 查当前天气和 7 天预报
metadata:
  {
    "openclaw": {
      "requires": {
        "env": ["WEATHER_API_KEY"]
      }
    }
  }
---

# 天气查询

## API 调用

```bash
# 当前天气
curl "https://api.weather.com/v1/current?location=beijing&key=$WEATHER_API_KEY"

# 7 天预报
curl "https://api.weather.com/v1/forecast?location=beijing&days=7&key=$WEATHER_API_KEY"
```

## 回复格式

```
北京天气
温度：25°C (体感 27°C)
湿度：65%
风向：东南风 3 级

未来 3 天：
- 明天：晴 23-30°C
- 后天：多云 22-28°C
- 大后天：小雨 20-25°C
```
```

### 3. 配置 Key

编辑 `~/.openclaw/openclaw.json`：

```json5
{
  skills: {
    entries: {
      "weather-pro": {
        enabled: true,
        env: {
          WEATHER_API_KEY: "你的 key"
        }
      }
    }
  }
}
```

### 4. 测试

```bash
openclaw gateway restart
# 然后问："北京今天天气怎么样？"
```
