## 🎯 核心概念：什么是技能？

在 OpenClaw 的世界里，**技能（Skills）** 是 Agent 的"超能力"——它们教会 AI 如何使用各种工具完成特定任务。

想象一下：
- 你的 Agent 突然学会了用 GitHub CLI 管理项目
- 它能调用天气 API 告诉你明天要不要带伞
- 它可以自动发送消息到 Telegram、Discord、飞书
- 甚至能控制浏览器自动完成复杂任务

这一切，都是通过**技能系统**实现的。

---

## 📁 技能系统的三层架构

OpenClaw 的技能加载遵循**三层优先级**设计：

```
┌─────────────────────────────────────┐
│  工作区技能 (workspace/skills)       │  ← 最高优先级
│  - 你的自定义技能                    │
│  - 针对当前项目定制                  │
├─────────────────────────────────────┤
│  本地管理技能 (~/.openclaw/skills)   │  ← 中等优先级
│  - 所有 Agent 共享                   │
│  - 覆盖内置技能                      │
├─────────────────────────────────────┤
│  内置技能 (bundled skills)           │  ← 基础优先级
│  - OpenClaw 自带                    │
│  - 版本更新时可能被覆盖              │
└─────────────────────────────────────┘
```

**优先级规则：** 工作区技能 → 本地管理技能 → 内置技能

这意味着你可以在工作区创建一个同名的技能，完全覆盖 OpenClaw 的默认行为。

## 🛠 实战：创建你的第一个技能

### 步骤 1：创建技能目录

```bash
# 在工作区的 skills 目录下创建新技能
mkdir -p ~/.openclaw/workspace/skills/hello-world
```

### 步骤 2：编写 SKILL.md 文件

每个技能的核心是一个 `SKILL.md` 文件，包含**YAML Frontmatter**和**Markdown 指令**：

```markdown
---
name: hello_world
description: 一个简单的问候技能，用于测试技能系统
metadata:
  {"openclaw": {"emoji": "👋"}}
---

# Hello World 技能

当用户要求打招呼、问候、或者说"你好"时，使用这个技能回复。

## 回复模板

- 标准问候："👋 你好！我是你的 AI 助手，有什么可以帮你的吗？"
- 早晨问候："☀️ 早上好！今天又是充满可能的一天！"
- 晚间问候："🌙 晚上好！希望你今天过得愉快！"

## 使用示例

用户说："打个招呼"
→ 回复："👋 你好！我是你的 AI 助手，有什么可以帮你的吗？"
```

### 步骤 3：测试技能

```bash
# 方法 1：重启 Gateway
openclaw gateway restart

# 方法 2：让 Agent 刷新技能
"请刷新一下技能列表"

# 方法 3：直接测试
openclaw agent --message "用 hello_world 技能打个招呼"
```

## 🔧 高级技能：定义自定义工具

真正的强大技能会**定义自己的工具**。让我展示一个完整的例子：

### 案例：GitHub 技能

这是 OpenClaw 工作区中真实存在的技能：

```markdown
---
name: github
description: 使用 gh CLI 与 GitHub 交互。用于 issues、PRs、CI runs 和高级查询。
---

# GitHub 技能

使用 `gh` CLI 与 GitHub 交互。始终在不处于 git 目录时指定 `--repo owner/repo`。

## Pull Requests

检查 PR 的 CI 状态：
```bash
gh pr checks 55 --repo owner/repo
```

列出最近的工作流运行：
```bash
gh run list --repo owner/repo --limit 10
```

查看运行记录并找出失败的步骤：
```bash
gh run view <run-id> --repo owner/repo
```

查看失败步骤的日志：
```bash
gh run view <run-id> --repo owner/repo --log-failed
```

## API 高级查询

`gh api` 命令可以访问其他子命令不可用的数据：

获取特定字段的 PR 信息：
```bash
gh api repos/owner/repo/pulls/55 --jq '.title, .state, .user.login'
```

## JSON 输出

大多数命令支持 `--json` 获取结构化输出，配合 `--jq` 过滤：

```bash
gh issue list --repo owner/repo --json number,title --jq '.[] | "\(.number): \(.title)"'
```
```

**关键点：**
- 技能通过**示例代码**教会 Agent 如何使用工具
- 提供**具体命令**而非抽象描述
- 包含**真实场景**的使用示例

## 🚪 技能门控：条件加载机制

OpenClaw 支持**智能加载**技能——只有满足条件时才会激活：

```markdown
---
name: nano-banana-pro
description: 通过 Gemini 3 Pro Image 生成或编辑图片
metadata:
  {
    "openclaw": {
      "requires": {
        "bins": ["uv"],
        "env": ["GEMINI_API_KEY"],
        "config": ["browser.enabled"]
      },
      "primaryEnv": "GEMINI_API_KEY",
      "emoji": "🎨"
    }
  }
---
```

**门控条件说明：**

| 条件类型 | 说明 | 示例 |
|---------|------|------|
| `bins` | 系统必须存在这些命令 | `["uv", "ffmpeg"]` |
| `anyBins` | 至少存在一个命令 | `["chrome", "brave", "edge"]` |
| `env` | 环境变量必须存在 | `["API_KEY", "TOKEN"]` |
| `config` | openclaw.json 配置必须为真 | `["browser.enabled"]` |
| `os` | 仅限特定操作系统 | `["darwin", "linux"]` |

**好处：**
- 避免技能因缺少依赖而报错
- 不同机器可以有不同的技能组合
- 敏感技能只在配置了 API Key 时激活

## ⚙️ 配置技能：openclaw.json

在 `~/.openclaw/openclaw.json` 中可以精细控制每个技能：

```json5
{
  skills: {
    entries: {
      // 通过技能名称引用
      "nano-banana-pro": {
        enabled: true,
        // API Key 管理（支持 SecretRef）
        apiKey: {
          source: "env",
          provider: "default",
          id: "GEMINI_API_KEY"
        },
        // 环境变量注入
        env: {
          GEMINI_API_KEY: "你的 API Key",
          ENDPOINT: "https://custom.api.com"
        },
        // 自定义配置
        config: {
          model: "nano-pro",
          timeout: 30000
        }
      },
      // 禁用某个技能
      "sag": {
        enabled: false
      },
      // 简单启用
      "peekaboo": {
        enabled: true
      }
    }
  }
}
```

**配置优先级：**
1. `env` 中的值只在变量不存在时注入
2. `apiKey` 是 `primaryEnv` 的快捷方式
3. `config` 用于技能自定义字段

## 🌐 ClawHub：技能市场

OpenClaw 有一个公共技能注册表——**ClawHub**（https://clawhub.com）

### 常用命令

```bash
# 搜索技能
clawhub search weather

# 安装技能到工作区
clawhub install weather

# 更新所有已安装的技能
clawhub update --all

# 同步（扫描并发布更新）
clawhub sync --all
```

### 技能发现策略

根据配置，技能查找顺序是：
1. **skillhub**（CN 优化，优先）
2. **clawhub**（公共注册表，备用）

## 📊 技能对 Token 消耗的影响

技能列表会注入到系统提示中，消耗一定的 Token：

**计算公式：**
```
总字符数 = 195 + Σ(97 + 技能名长度 + 描述长度 + 路径长度)
```

**估算：**
- 基础开销（至少 1 个技能）：195 字符
- 每个技能：约 97 字符 + 实际内容
- 按 OpenAI 分词器：~4 字符/Token
- **每个技能约 24 Token**

**优化建议：**
- 保持技能描述简洁
- 合并功能相似的技能
- 禁用不常用的技能

## 🔒 安全注意事项

技能本质上是**给 AI 的指令**，但可能涉及敏感操作：

### 风险点

1. **第三方技能是未受信任的代码**
   - 始终先阅读 SKILL.md 再启用
   - 检查是否有可疑的 shell 命令

2. **环境变量注入**
   - `skills.entries.*.env` 会注入到**主机进程**
   - 不要在提示词中泄露密钥

3. **沙箱逃逸**
   - 沙箱中的技能需要容器内也有对应二进制文件
   - 通过 `agents.defaults.sandbox.docker.setupCommand` 安装

### 最佳实践

```markdown
# ✅ 好的做法
- 明确指定命令参数，避免任意命令注入
- 使用 `--repo owner/repo` 明确目标
- 提供错误处理示例

# ❌ 避免的做法
- 执行用户输入的任意命令
- 在技能中硬编码 API Key
- 不加验证地调用外部 API
```

## 🎓 实战项目：创建一个天气技能

让我们从零开始创建一个完整的天气技能：

### 1. 创建目录结构

```bash
mkdir -p ~/.openclaw/workspace/skills/weather-pro
cd ~/.openclaw/workspace/skills/weather-pro
```

### 2. 编写 SKILL.md

```markdown
---
name: weather-pro
description: 获取当前位置和未来 7 天天气预报
metadata:
  {
    "openclaw": {
      "requires": {
        "env": ["WEATHER_API_KEY"]
      },
      "primaryEnv": "WEATHER_API_KEY",
      "emoji": "🌤"
    }
  }
---

# 天气查询技能

使用天气 API 获取当前位置和未来 7 天的天气预报。

## API 端点

```bash
# 当前天气
curl "https://api.weather.com/v1/current?location={location}&key=$WEATHER_API_KEY"

# 7 天预报
curl "https://api.weather.com/v1/forecast?location={location}&days=7&key=$WEATHER_API_KEY"
```

## 使用示例

### 查询北京当前天气

```bash
curl "https://api.weather.com/v1/current?location=beijing&key=$WEATHER_API_KEY" | jq '.'
```

### 查询上海 7 天预报

```bash
curl "https://api.weather.com/v1/forecast?location=shanghai&days=7&key=$WEATHER_API_KEY" | jq '.'
```

## 回复格式

将 API 响应转换为用户友好的格式：

```
🌤 北京天气
温度：25°C (体感 27°C)
湿度：65%
风向：东南风 3 级
空气质量：良

📅 未来 3 天预报
- 明天：晴 23-30°C
- 后天：多云 22-28°C
- 大后天：小雨 20-25°C
```

## 错误处理

如果 API 调用失败：
1. 检查 API Key 是否有效
2. 检查网络连接
3. 返回友好的错误提示："暂时无法获取天气信息，请稍后再试"
```

### 3. 配置 API Key

编辑 `~/.openclaw/openclaw.json`：

```json5
{
  skills: {
    entries: {
      "weather-pro": {
        enabled: true,
        env: {
          WEATHER_API_KEY: "你的天气 API Key"
        }
      }
    }
  }
}
```

### 4. 测试技能

```bash
# 重启 Gateway 使配置生效
openclaw gateway restart

# 测试
"北京今天天气怎么样？"
"上海未来一周会下雨吗？"
```

## 🚀 技能开发最佳实践

### 1. 命名规范

```bash
# ✅ 好的命名
weather-pro          # 清晰描述功能
github-advanced      # 表明是增强版
moltbook-publisher   # 完整描述用途

# ❌ 避免的命名
skill1              # 无意义
my-skill            # 太泛
test                # 临时名称
```

### 2. 描述撰写

```markdown
# ✅ 好的描述
- "使用 gh CLI 管理 GitHub issues 和 PRs"
- "通过 CDP 控制 Chrome 浏览器自动登录"
- "调用飞书 API 发送消息和创建文档"

# ❌ 避免的描述
- "一个很有用的技能"（太模糊）
- "做各种事情"（不具体）
- "自动化工具"（缺少上下文）
```

### 3. 示例代码

- 提供**可直接复制**的命令
- 包含**真实参数**而非占位符
- 展示**成功和失败**两种场景

### 4. 错误处理

```markdown
## 常见问题

### API Key 无效
检查 `~/.openclaw/openclaw.json` 中的配置

### 命令不存在
运行 `which <command>` 确认安装

### 权限不足
检查沙箱配置和 Exec 审批设置
```
