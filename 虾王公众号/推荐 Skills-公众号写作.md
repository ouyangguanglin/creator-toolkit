## 💡 自定义 Skill 开发

如果现有 Skills 不能满足需求，可以开发自己的 Skill：

### Skill 结构
```
my-writing-skill/
├── SKILL.md          # 技能说明
├── package.json      # 元数据
├── index.js          # 主逻辑
└── README.md         # 使用文档
```

### 发布到 ClawHub
```bash
clawhub publish ./my-writing-skill
```

## 📊 Skills 使用建议

### ✅ 最佳实践
1. **按需安装** - 不要一次性安装太多，用得上再装
2. **定期更新** - `clawhub update` 保持最新版本
3. **查看文档** - 每个 Skill 的 SKILL.md 都有详细说明
4. **组合使用** - 多个 Skills 配合效果更佳

### ❌ 避免什么
1. **不要重复** - 功能相似的 Skill 选一个就好
2. **不要忽略权限** - 检查 Skill 需要的权限
3. **不要跳过测试** - 新 Skill 先小范围测试
4. **不要忽略日志** - 出错了查看日志好调试
